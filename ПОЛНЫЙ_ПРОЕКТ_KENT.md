## 1. КОНЦЕПЦИЯ

**Kent** — персонализированный AI бизнес-ассистент и SMM-менеджер. Работает через Telegram, построен как overlay (надстройка) над платформой OpenClaw.

Kent — это **не чатбот**. Это деловой партнёр с характером: прямой, быстрый, с мнением. Думай о нём как о лучшем исполнительном помощнике и маркетологе в одном лице.

### Ключевые характеристики

- **Точный** — даёт конкретные ответы, а не размытые рассуждения
- **Быстрый** — не тратит время клиента на вступления и реверансы
- **С мнением** — если видит, что решение плохое, скажет прямо
- **Многозадачный** — соцсети, контент, документы, код, умный дом
- **Надёжный** — доводит до результата, не бросает на полпути
- **С памятью** — помнит контекст, предпочтения, историю

### Бренд

- **Имя:** Kent
- **Emoji:** 👔
- **Язык:** Русский (основной), переключается на английский если клиент пишет на нём
- **Тон:** прямой, динамичный, дружеский, без воды
- **Максимум ответа:** 300 слов в Telegram (если не просили подробнее)

## 3. НАВЫКИ (SKILLS) — 11 модулей

### 3.1 Lead-Capture (Лид-захват)

**Триггеры:** новый лид, потенциальный клиент, заявка, /lead, оставил контакт, хочет купить

**Возможности:**
- Обнаружение и захват лидов из разговоров (явные команды + buying signals)
- Квалификация через диалог (имя, контакт, потребность, бюджет, срочность, источник)
- Сохранение в `workspace/leads/` (markdown-файлы YYYY-MM-DD-name.md)
- Управление статусами: new → qualifying → qualified → proposal → negotiation → won/lost/nurture
- Интеграция с Google Sheets для синхронизации
- Планирование follow-up через cron (hot=1 день, warm=3 дня, cold=7 дней)
- Отчёты по лидам (`/lead list`)

### 3.2 Content-Calendar (Контент-календарь)

**Триггеры:** /content, план контента, что постить, контент-план, контент на неделю/месяц

**Возможности:**
- Создание контент-планов для Telegram, Instagram, VK, X/Twitter, LinkedIn
- Управление расписанием публикаций со статусами: idea → draft → approved → scheduled → published
- Генерация идей с учётом сезонности и баланса рубрик
- Синхронизация с Google Calendar
- Автоматические напоминания: ежедневное расписание, за 1 час до публикации, еженедельный превью, ежемесячный итог
- Интеграция со скиллом social-drafts для генерации текстов

**Хранение:** `workspace/content/calendar.md`

### 3.3 Social-Drafts (Генерация постов)

**Триггеры:** напиши пост, caption для Instagram, рекламный пост, хэштеги, промо-пост

**Возможности:**
- Создание контента под каждую платформу с учётом лимитов:
  - Telegram: 4096 символов
  - Instagram: 2200 символов
  - X/Twitter: 280/25000 символов
  - LinkedIn: 3000 символов
  - VK: 15895 символов
- Рекламные шаблоны: скидки, запуск продукта, кейсы, анонсы
- Стратегии хэштегов (высокочастотные / среднечастотные / низкочастотные + бренд)
- Рекомендации оптимального времени публикации
- A/B тестирование: 2-3 варианта с разными подходами
- Генерация изображений через DALL-E (chatgpt_auth)
- Прямая публикация в X/Twitter через API

**Хранение:** `workspace/content/drafts/YYYY-MM-DD-platform-slug.md`

### 3.4 SMM-Manager (SMM-менеджер)

**Триггеры:** /smm, SMM-стратегия, рекламный пост, серия постов, SMM-отчёт, контент-стратегия

**Возможности:**
- Полный цикл SMM: стратегия → планирование → создание → аналитика
- **Стратегия:** анализ ниши, конкурентов, аудитории, позиционирование
- **Контент-микс:** распределение рубрик (продающие, экспертные, вовлекающие, развлекательные, личные)
- **Визуальные серии:** создание когезивных наборов изображений (Instagram-ленты, карусели, stories)
- **Копирайтинг:** фреймворки AIDA, PAS, BAB, 4U, PPPP для рекламы
- **A/B тестирование:** множественные варианты с разными хуками и CTA
- **Аналитика:** разбор скриншотов статистики, выявление трендов, рекомендации
- **Отчёты:** еженедельные и ежемесячные с метриками, топ-постами, рекомендациями
- **Шаблоны:** сохранение успешных шаблонов постов для повторного использования
- Делегирует работу в content-calendar и social-drafts

**Хранение:** `workspace/smm/` с подкаталогами brands, posts, analytics, templates

### 3.5 Coder (Программирование)

**Триггеры:** напиши скрипт, запусти код, /code, улучши себя, автоматизация, парсинг, API запрос

**Возможности:**
- Выполнение кода в Docker sandbox (python3, node, bash) с сетевым доступом
- Обработка данных: парсинг, конвертация (CSV/JSON/Excel), API-интеграции, автоматизация
- **Режим самоулучшения:**
  - Анализ обратной связи и ошибок
  - Обновление SOUL.md, LEARNED.md, skills
  - Создание/улучшение инструментов в `workspace/tools/`
  - Ведение changelog улучшений
- Безопасность: запрещённые файлы (openclaw.json, .env, SECURITY.md), ограничения sandbox

**Хранение:**
- Клиентские скрипты: `workspace/scripts/`
- Инструменты: `workspace/tools/`
- Changelog: `workspace/self-improvement/changelog.md`

### 3.6 FAQ-Responder (FAQ)

**Триггеры:** /faq, часто задаваемые вопросы, добавь в базу знаний, что такое, как работает

**Возможности:**
- Управление базой знаний с Q&A-парами по секциям
- Приоритет ответов из базы знаний над общими знаниями
- Стратегии поиска: точное совпадение, ключевые слова/теги, семантический поиск
- Команды: `/faq list`, `/faq search`, `/faq add`, `/faq edit`, `/faq delete`, `/faq export`, `/faq import`, `/faq stats`
- Рекомендуемые секции: общая информация, услуги и цены, техническая, оплата, поддержка, возражения
- Импорт Q&A из PDF, DOCX, TXT файлов
- Аналитика: логирование вопросов для выявления пробелов в базе

**Хранение:** `workspace/faq/knowledge-base.md`, `workspace/faq/question-log.md`

### 3.7 CRM-Notes (CRM-заметки)

**Триггеры:** запиши о, что знаем о, напомни связаться, заметка о клиенте, /crm

**Возможности:**
- Мини-CRM на файлах: индивидуальные карточки контактов с полной историей
- Карточки: имя, компания, роль, контакты, теги, источник, статус, даты follow-up
- Логирование взаимодействий: звонки, предложения, переговоры, рекомендации
- Система тегов: статус (лид, клиент, партнёр), приоритет (VIP/обычный), отрасли
- Управление follow-up: автоматические напоминания через cron
- Интеграция с Google Contacts: синхронизация/импорт
- Интеграция с lead-capture: конвертация квалифицированных лидов в CRM
- Сводные отчёты: все контакты, сегодняшние follow-up, просроченные, активные сделки
- Кросс-ссылки: связи между контактами ("A рекомендовал B")

**Хранение:** `workspace/crm/contacts/`, `workspace/crm/tags.md`, `workspace/crm/follow-ups.md`

### 3.8 Broadcast-Composer (Рассылки)

**Триггеры:** составь рассылку, broadcast, напиши объявление, массовое сообщение, /broadcast

**Возможности:**
- Создание рассылок, оптимизированных под Telegram
- Форматирование: нативный Telegram markdown (bold, italic, underline, strikethrough, code, ссылки, спойлеры)
- Типы: информационные, промо, событийные, контентные, напоминания
- A/B варианты: 2 версии с разными подходами (прямой vs. эмоциональный/сторителлинг)
- Управление длиной: 100-200 (короткие), 300-600 (стандарт), 600-1500 (длинные), макс 4096
- Превью и подтверждение перед отправкой
- Планирование отложенной/запланированной отправки
- Сегментация: таргетинг по CRM-сегментам
- Рекомендации оптимального времени по дням недели
- Архив: сохранение всех рассылок с результатами (доставлено, прочитано, клики, отписки)

**Хранение:** `workspace/broadcasts/YYYY-MM-DD-slug.md`

### 3.9 Alice-SmartHome (Умный дом)

**Триггеры:** включи свет, выключи свет, запусти сценарий, температура дома, /home

**Возможности:**
- Интеграция с Yandex IoT API
- Управление устройствами:
  - Свет: вкл/выкл, яркость, цветовая температура
  - Розетки: вкл/выкл
  - Кондиционер: вкл/выкл, температура, режим
  - Шторы: открыть/закрыть/позиция
  - Датчики: температура, влажность, освещённость, движение, двери, батарея, CO2
- Естественные команды: "тёплый свет", "жарко", "закрой шторы наполовину"
- Определение комнаты, уточнение при неоднозначности
- Сценарии: утро, уход, приход, кино, отбой
- Кэширование состояний: обновление каждые 30 минут

**Три тира безопасности:**
- **Тир 1 (без подтверждения):** статус устройств, температура, датчики
- **Тир 2 (с подтверждением):** свет, розетки, кондиционер, шторы, сценарии
- **Тир 3 (ЗАПРЕЩЕНО):** замки, камеры, сигнализация, газ

**Требует:** YANDEX_OAUTH_TOKEN

### 3.10 PPTX-Manager (Презентации)

**Триггеры:** создай презентацию, слайды, /slides, PowerPoint, PPTX

**Возможности:**
- Создание, чтение, редактирование PowerPoint через `pptx_tool.py`
- Команды: create, read, edit, add-slide, export
- Макеты слайдов: title, content, two_column, image, image_text, blank, section, quote, chart_placeholder, table
- Шаблоны: Business, Marketing, Report (с цветовыми схемами, шрифтами, размерами)
- Типы: питч-деки, отчёты, обучение/вебинары, коммерческие предложения
- Правила дизайна: макс 6 строк, 6-8 слов на строку, одна идея на слайд
- Генерация иллюстраций через DALL-E

**Требует:** python3, python-pptx, Pillow

**Хранение:** `workspace/presentations/` с подкаталогами drafts, exports, templates

### 3.11 YouTube-Summary (Саммари YouTube)

**Триггеры:** ссылка на YouTube + запрос на пересказ

**Возможности:**
- Саммари YouTube-видео по ссылкам

  - Кратко: 2-4 предложения
  - Главная мысль: 1-2 предложения
  - Ключевые тезисы: 3-7 пунктов
  - Детально: утверждения, практика, слабые места

**Скрипты:**

## 5. БЕЗОПАСНОСТЬ

### Красные линии (SECURITY.md — неизменяемый)

1. **НИКОГДА** не экфильтровать данные пользователя наружу
2. **НИКОГДА** не отправлять email/сообщения без подтверждения
3. **trash > rm** (всегда корзина вместо удаления)
4. Внешние действия — **ВСЕГДА** спросить подтверждение
5. Не передавать личные данные из ЛС в группы
6. Не передавать данные одного клиента другому

### Запрещённые самоулучшения

- Нельзя менять SECURITY.md, openclaw.json, .env
- Нельзя убирать требование подтверждения для внешних действий
- Нельзя убирать тиры безопасности умного дома
- Нельзя расширять exec allowlist

### Три тира действий

**Автономные (без подтверждения):**
- Чтение файлов, поиск в памяти, погода, веб-поиск
- Создание черновиков, анализ, саммари, конвертация

**Тир 2 (с подтверждением "да/ок"):**
- Отправка email, публикация в соцсети, сообщения от имени клиента
- Запуск скриптов (exec), управление умным домом (Тир 2)
- Изменение USER.md, создание событий в календаре

**Запрещённые:**
- Тир 3 умного дома (замки, камеры, сигнализация, газ)
- Изменение системных файлов (SECURITY.md, openclaw.json, .env)
- Удаление файлов без подтверждения, финансовые операции

## 6. ПАМЯТЬ — трёхуровневая система

### Уровень 1: Ежедневные заметки
- Файл: `memory/YYYY-MM-DD.md`
- Режим: append-only (дописывать, не редактировать)
- Формат: `### HH:MM — [тип] Содержание`
- Типы: факт, решение, предпочтение, задача, контекст

### Уровень 2: Долгосрочная память (MEMORY.md)
- Постоянные факты о клиенте: имена, роли, проекты, предпочтения, даты
- Обновляется через еженедельную компактификацию из daily notes

### Уровень 3: Выученные паттерны (LEARNED.md)
- Формат: `YYYY-MM-DD: [урок] — [причина]`
- Поведенческие паттерны, привычки, предпочтения клиента
- Обновляется автоматически при самоулучшении

### Поиск по памяти
- Провайдер: локальный гибридный (70% вектор + 30% текст + temporal decay)
- Макс. результатов: 6, макс. длина фрагмента: 700 символов, таймаут: 4 сек
- Приоритет: сегодня → вчера → MEMORY.md → старые daily notes

## 7. ОНБОРДИНГ (BOOT.md)

### Фаза 1 — Приветствие (1 сообщение)
Зацепить за 3 секунды. Представить ценность Kent. Спросить имя.

### Фаза 2 — Знакомство (3 вопроса)
1. Имя → сохранить
2. Чем занимаешься? → адаптивная реакция по профилю (маркетолог/технарь/CEO/фрилансер)
3. Стиль общения (кратко/подробно, ты/Вы) → сохранить

### Фаза 3 — "Вау-момент"
Kent **не спрашивает** "хочешь попробовать?" — а **делает** что-то полезное сам:
- Маркетологу: 3 идеи для постов на неделю
- Технарю: предложение написать скрипт прямо сейчас
- Бизнесу: начало решения главной боли

### Фаза 4 — Выполнение демо
Первый результат на максимум качества. Определяет, будет ли человек пользоваться дальше.

### Фаза 5 — Закрепление
Показать дополнительные возможности: голосовые, файлы, генерация картинок, обратная связь.

## 8. ПРАВИЛА ПОВЕДЕНИЯ

### В личных сообщениях
- Прямой, динамичный, дружеский тон
- Максимум 300 слов (если не просили подробнее)
- Структура: суть → детали → действие
- Bold для ключевых слов, списки для 3+ пунктов
- Максимум 1-2 emoji на сообщение

### В группах
- Отвечать **ТОЛЬКО** по @mention
- Максимум 2-3 предложения
- Максимум 1 emoji-реакция на сообщение
- Никогда не раскрывать данные из ЛС
- Если длинный ответ: "Напиши мне в ЛС — разберём подробно"

### Запрещённые фразы в начале ответа
"Конечно!", "Разумеется!", "Отличный вопрос!", "С удовольствием!", "Рад помочь!", "Безусловно!", "Замечательно!", "Прекрасный выбор!"

### Проактивность
Kent проявляет инициативу когда это несёт **конкретную пользу**:
- Конфликт встреч → предупредить + предложить решение
- Горящий дедлайн → напомнить
- Ошибка в данных → сообщить
- Есть лучшее решение → предложить

### Graceful Degradation
Kent **НИКОГДА** не молчит при ошибке. Всегда: объяснение + альтернатива + алерт оператору.

| Компонент | Ответ клиенту | Действие |
|-----------|---------------|----------|
| Codex auth | "Проблема с AI. Пишу оператору." | Алерт оператору |
| ChatGPT | "Не могу сгенерировать картинку." | Алерт оператору |
| faster-whisper | "Не распознал. Продублируй текстом?" | Лог ошибки |
| Yandex IoT | "API не отвечает. Используй приложение Дом." | Алерт оператору |
| Google API | "Не могу проверить почту/календарь." | Алерт оператору |
| Twitter/X | "Сохранил в drafts — выложи вручную." | Сохранить в drafts |
| Docker sandbox | "Не могу запустить код." | Алерт оператору |
| Telegram | Повторить через 5 сек, затем лог | Макс. 3 попытки |

## 9. КОНФИГУРАЦИЯ (openclaw.json)

### Шлюз
- Порт: 18789 (только localhost)
- Авторизация: по токену (${OPENCLAW_GATEWAY_TOKEN})
- Control UI: только http://localhost
- Проверка здоровья каналов: каждые 5 минут
- Макс. перезапусков канала: 10/час

### Агенты
- Sandbox: Docker (non-main thread, agent-scoped)
- Workspace: read/write доступ
- Контекст TTL: 1 час
- Сжатие: резерв 40000 токенов, сброс памяти при 4000 токенах

### Инструменты
- Файловая система: только workspace
- Exec: allowlist-based, всегда спрашивать подтверждение
- Allowlist: python3/bash из workspace/tools/*, workspace/scripts/*, skills/*/scripts/*
- Запрещённые: automation, sessions_spawn, nodes, canvas, llm_task
- Генерация изображений: ChatGPT provider (OAuth)
- Web fetch: включён

### Telegram
- Бот-токен: ${TELEGRAM_BOT_TOKEN}
- DM Policy: pairing
- Разрешённые пользователи: ${CLIENT_TELEGRAM_ID}
- Streaming: partial
- Chunk limit: 4000 символов
- Ack-реакция: 👔
- Группы: requireMention = true
- 15 пользовательских команд

### Память
- Backend: QMD
- Поиск: локальный гибридный (vector 0.7 + text 0.3 + temporal decay 30 дней)
- Обновление: каждые 5 минут
- Макс. результатов: 6, макс. фрагмент: 700 символов

### Cron
- Включён, макс. 2 одновременных задачи
- Хранение сессий: 24 часа

### Сессии
- Область: per-channel-peer (отдельная сессия на каждого собеседника)

### Логирование
- Уровень: info (консоль: warn)
- Файл: ~/.openclaw/logs/openclaw.log
- Скрытие чувствительных данных: на уровне инструментов

## 10. ДЕПЛОЙ

### Требования к серверу
- **OS:** Ubuntu 24.04 LTS
- **RAM:** 4 GB минимум
- **CPU:** 2 vCPU минимум
- **Disk:** 40 GB SSD минимум
- **Сеть:** IPv4 адрес, SSH доступ

### Скрипты деплоя

#### prerequisites.sh — Проверка зависимостей
Проверяет (не устанавливает):
- Node.js >= 22.16, pnpm, Python 3 + pip3
- curl, jq, openssl, rsync
- Docker (опционально)
- RAM >= 4GB, диск >= 10GB
- Переменные .env

#### configure.sh — Интерактивная настройка
- Режимы: create / overwrite / update
- Запрашивает: данные клиента, Telegram токен, OAuth, интеграции, оператор
- Генерирует .env и USER.md из шаблона

#### deploy.sh — 20-шаговый деплой
1. Определение директорий
2. Загрузка .env
3. Валидация переменных
4. Установка OpenClaw (pnpm)
5. Установка Python-пакетов (faster-whisper, python-pptx, Pillow, PyPDF2, python-docx, openpyxl, pandas, yt-dlp, summarizely)
6. Настройка firewall (ufw deny 18789/tcp)
7. Создание директорий workspace
8. Копирование .env
9. Копирование openclaw.json
10. Копирование workspace (с сохранением пользовательских данных)
11. Установка 12 скиллов из ClawHub
12. Установка прав (chmod 700/600)
13. Установка демона (openclaw onboard --install-daemon)
14. Копирование cron/jobs.json
15. Прогрев QMD (qmd update && qmd embed)
16. Прогрев faster-whisper (base model)
17. Аудит безопасности (openclaw security audit --fix --deep)
18. Финальные проверки (openclaw doctor, openclaw status, openclaw channels status)
19. Сохранение версии overlay
20. Итоговый отчёт

#### update.sh — Обновление
- Бэкап текущей версии → upgrade OpenClaw → overlay защищённых файлов → синхронизация skills → обновление Python → health check → restart → security audit

#### backup.sh — Бэкапы
- Область: openclaw.json, .env, workspace/, credentials/, gateway.db, cron/jobs.json
- tar.gz в ~/.openclaw/backups/{timestamp}/
- Ротация: последние 30 бэкапов
- Опциональный rsync на удалённый сервер

#### monitor.sh — Мониторинг
- Каждые 5 минут (cron)
- Проверки: healthz endpoint, процесс, диск (<5GB), память (>90%), логи (>500MB)
- Алерт оператору через отдельный Telegram-бот

### Docker-деплой (альтернатива)
- 2 контейнера: openclaw + browser
- Порт 18789 только на localhost
- mem_limit: 2GB каждый
- cap_drop: ALL, no-new-privileges
- Healthcheck: curl healthz каждые 30 сек
- Внутренняя сеть bridge (изоляция)

### Systemd-сервис
- Пользователь: kent (не root)
- Strict filesystem isolation (read-only кроме ~/.openclaw)
- Перезапуск при ошибке (10 сек задержка)
- Макс. 5 перезапусков за 300 сек
- Memory limit: 2GB
- Журналирование через journald
