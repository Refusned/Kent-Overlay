# ТЗ для подрядчика за 3 минуты

## Для кого
Предприниматели, менеджеры проектов — кому нужно быстро и чётко поставить задачу исполнителю.

## Что нужно подключить
- Базовый Kent (текстовый AI) — обязательно

## Промпт

```
Составь техническое задание для подрядчика.

Что нужно сделать: [например: разработать лендинг для курсов английского]
Тип подрядчика: [например: веб-дизайнер / разработчик / копирайтер / SMM-менеджер]
Бюджет: [например: до 50 000 руб]
Срок: [например: 2 недели]
Референсы: [например: похож на skyeng.ru, но проще]

Структура ТЗ:
1. Описание проекта (2-3 предложения)
2. Цели и задачи (что должно получиться)
3. Целевая аудитория (для кого делаем)
4. Функциональные требования (что должно работать)
5. Технические требования (платформа, хостинг, интеграции)
6. Контент (что предоставляет заказчик, что делает подрядчик)
7. Этапы и сроки (разбивка на этапы с датами)
8. Критерии приёмки (как понять, что работа сделана)
9. Бюджет и порядок оплаты (предоплата / по этапам / по факту)

Формат: готовый документ, который можно скопировать и отправить подрядчику.
```

## Как настроить под себя
- Для дизайнера добавьте: "Включи раздел про стиль: цвета, шрифты, настроение"
- Для разработчика: "Включи требования к производительности и безопасности"
- Для копирайтера: "Включи tone of voice и примеры текстов, которые нравятся"

## Пример результата
Структурированное ТЗ на 1-2 страницы с конкретными требованиями, сроками, критериями приёмки. Готово для отправки на фриланс-биржу или подрядчику.
