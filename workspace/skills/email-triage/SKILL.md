# 📬 Email Triage — Триаж почты

Скилл для управления электронной почтой через Telegram. Категоризация, приоритизация, черновики ответов — без открывания Gmail.

## Возможности

### 1. Утренний дайджест
По команде "проверь почту" или автоматически (cron — каждые 2 часа):
- Получить непрочитанные письма через Gmail API
- Категоризировать:
  - 🔴 **Срочно** — требует ответа сегодня (есть дедлайн, вопрос, запрос на действие)
  - 🟡 **Ответить** — не срочно, но нужен ответ (обсуждения, уточнения)
  - 🔵 **Инфо** — для ознакомления (рассылки, уведомления, подтверждения)
  - ⚪ **Пропустить** — спам, маркетинг, неважное
- Формат вывода:
  ```
  📬 Дайджест почты (12 новых)

  🔴 Срочно (2):
  1. Иванов И.И. — "Договор на подпись" (вчера 18:30) → нужна подпись до пятницы
  2. ООО Ромашка — "Оплата просрочена" (сегодня 09:15) → счёт №45, дедлайн сегодня

  🟡 Ответить (3):
  3. Петров П. — "Уточнение по ТЗ" → спрашивает про сроки
  4. Сидорова А. — "Re: Встреча" → предлагает перенести на четверг
  5. Марков Д. — "Коммерческое предложение" → просит рассмотреть

  🔵 Инфо (4): Google Calendar, Wildberries, банк, подписка
  ⚪ Пропустить (3): маркетинг

  Что сделать? Напишите номер письма для подробностей или "ответь на 1".
  ```

### 2. Подробности письма
По номеру из дайджеста — показать:
- От кого, когда, тема
- Краткое содержание (3-5 предложений, НЕ полный текст)
- Вложения (если есть)
- Рекомендуемое действие

### 3. Черновик ответа
По команде "ответь на [номер]":
- Сгенерировать черновик ответа на основе контекста письма
- Учитывать стиль пользователя (из SOUL.md)
- Показать черновик для подтверждения
- После подтверждения — отправить через Gmail API или сохранить как черновик

### 4. Поиск по почте
"Найди письмо от Иванова про договор":
- Поиск через Gmail API (from:, subject:, has:attachment)
- Вывод: отправитель, дата, тема, первые строки
- До 5 результатов

### 5. Автоматический мониторинг (cron)
При настройке cron-задачи:
- Каждые 2 часа проверять новые письма
- Уведомлять только о 🔴 (срочных)
- Формат: "📬 Новое срочное: [отправитель] — [тема]"

## Безопасность

- НИКОГДА не показывать полный текст письма (только краткое содержание)
- НИКОГДА не пересылать письма без подтверждения
- НИКОГДА не отправлять ответ без явного подтверждения пользователя
- Пароли, токены, персональные данные из писем — не показывать в дайджесте
- В групповых чатах: скилл НЕ активируется (только в личных сообщениях)

## Обработка ошибок

- Gmail API не отвечает → "Не удалось подключиться к почте. Проверьте настройки Google в /settings"
- Нет новых писем → "📬 Ничего нового! Входящие пусты."
- OAuth токен истёк → "Токен Gmail истёк. Обновите через /settings → Google"

## Зависимости

- Требует настроенной интеграции Google (gog) с разрешением на Gmail
- Использует Gmail API: messages.list, messages.get, messages.send, drafts.create
