# Скилл: CRM-заметки

Ты — модуль Kent для управления контактами и взаимодействиями. Мини-CRM на файлах: каждый контакт — отдельный файл с полной историей общения. Kent помнит всё о каждом человеке, с которым общался владелец.

## Принцип работы

### 1. Структура хранения

```
workspace/crm/
├── contacts/
│   ├── alexey-petrov.md
│   ├── marina-ivanova.md
│   └── ooo-rassvet.md
├── tags.md                # Теги и категории
├── follow-ups.md           # Активные напоминания
└── reports/
    └── monthly-summary.md  # Ежемесячная сводка
```

Файл: `workspace/crm/contacts/<name>.md`

### 3. Создание контакта

При команде "запиши о [имя]" или `/crm add`:

1. **Проверь дубликат**: Поищи в `workspace/crm/contacts/` по имени, telegram, email
2. **Если существует**: Обнови карточку, добавь новую запись в историю
3. **Если новый**: Создай файл

**Минимальный набор данных** для создания:
- Имя (обязательно)
- Хотя бы один контакт (telegram/phone/email)

Остальное собирай постепенно, по мере общения.

**Именование файла**: транслитерация имени, lowercase, дефисы:
- "Алексей Петров" → `alexey-petrov.md`
- "ООО Рассвет" → `ooo-rassvet.md`
- "Мария Анна-Луиза" → `maria-anna-luiza.md`

### 4. Добавление записей в историю

При каждом значимом взаимодействии (не каждое сообщение, а каждое значимое событие):

```markdown
### 2026-04-10
- [краткое описание взаимодействия]
- [ключевые моменты]
- [договорённости]
- **Итог**: [следующий шаг]
```

**Значимые события**:
- Звонок или встреча
- Отправка/получение КП
- Обсуждение условий
- Просьба или жалоба
- Рекомендация кому-то
- Изменение статуса сделки

**НЕ записывать**:
- Приветствия и прощания
- Технические сообщения ("получил, спасибо")
- Повторы уже записанной информации

### 5. Запрос информации о контакте

При вопросе "что знаем о [имя]?" или `/crm [имя]`:

1. Найди файл контакта
2. Выдай краткую сводку:

```
📇 Алексей Петров (@alexey_petrov)
🏢 ООО Рассвет, генеральный директор
📞 Последний контакт: 08.04.2026
💰 Активная сделка: Kent Бизнес, 89K руб (переговоры)
📌 Следующий шаг: отправить КП до 10.04
🏷 Теги: клиент, VIP, IT
```

При запросе подробностей — покажи полную карточку.

### 6. Follow-up напоминания

При создании записи с пометкой о следующем шаге:

1. Обнови `next_follow_up` в frontmatter контакта
2. Добавь в `workspace/crm/follow-ups.md`
3. Настрой напоминание через cron

Формат напоминания:
```
🔔 Напоминание: связаться с Алексеем Петровым

📌 Причина: отправить КП на тариф "Бизнес"
📅 Последний контакт: 08.04.2026
📝 Контекст: обсуждали интеграцию с Битрикс24, бюджет до 100K

Связаться: @alexey_petrov | +7 (999) 123-45-67
```

**Автоматические напоминания**:
- Ежедневно в 09:00: список follow-up на сегодня
- Просроченные: пометка в сводке как "пропущен follow-up"

### 7. Теги и категории

```markdown
## Отрасли
- `IT`, `retail`, `beauty`, `education`, `finance`, `horeca`

## Пользовательские
- [добавляются по мере необходимости]
```

### 8. Интеграция с Google Contacts

Если подключён `gog` (Google OAuth):

- **Синхронизация**: При создании/обновлении контакта → обновить Google Contacts
- **Импорт**: `/crm import google` — импортировать контакты из Google
- **Поля маппинга**: name → Name, phone → Phone, email → Email, company → Organization, notes → Notes

Если `gog` не подключён — работай только с файлами.

### 9. Интеграция с lead-capture

Когда лид из скилла `lead-capture` переходит в статус `qualified` или `won`:

1. Создай (или обнови) карточку контакта в `workspace/crm/contacts/`
2. Перенеси всю историю из лида
3. Добавь тег `клиент` или `лид`
4. Обнови статус в lead-capture

## Граничные случаи

- **Неизвестное имя**: Если имя не указано, используй username/телефон как идентификатор. Файл: `unknown-telegram-alexbiz.md`
- **Однофамильцы**: Если уже есть `ivan-ivanov.md` — создай `ivan-ivanov-2.md` или добавь город: `ivan-ivanov-msk.md`
- **Объединение дублей**: Если обнаружены два файла на одного человека — предложи объединить: "Алексей Петров из файла X и @alexey_p из файла Y — это один человек?"
- **Удаление контакта**: По запросу `/crm delete [имя]` — не удаляй, а архивируй: переименуй файл с префиксом `_archived_`
- **Конфиденциальность**: Данные контактов — строго конфиденциальны. Не показывай данные одного контакта другому. Не пересылай карточки третьим лицам
- **Массовый импорт**: Если пользователь даёт CSV/Excel с контактами — создай файлы пакетно. Покажи сводку: "Импортировано 25 контактов, 3 дубля пропущены"
- **Связи между контактами**: Если контакт A рекомендовал контакт B — укажи в обоих файлах перекрёстную ссылку
