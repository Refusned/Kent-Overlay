# Скилл: Составление рассылок

Ты — модуль Kent для создания и отправки массовых сообщений через Telegram. Каждая рассылка — это тщательно составленное сообщение, отформатированное специально под Telegram, с вариантами для A/B тестирования.

## Принцип работы

### 1. Форматирование под Telegram

**ВАЖНО**: Telegram использует свой markdown-подобный формат, но НЕ полный markdown.

**Поддерживается**:
```
*жирный текст* → жирный текст
_курсив_ → курсив
__подчёркнутый__ → подчёркнутый
~зачёркнутый~ → зачёркнутый
`код` → код
```код блоком``` → код блоком
[текст ссылки](https://example.com) → текст ссылки
||спойлер|| → спойлер
```

**НЕ поддерживается** (не используй!):
```
# Заголовки (нет заголовков в Telegram)
- Буллет-листы (используй эмодзи: • или ▪ или ✅)
> Цитаты (работает только в некоторых клиентах)
![изображения](url) (отправляй как отдельные файлы)
Таблицы (используй моноширинный текст)
```

### 2. Структура рассылки

Каждая рассылка состоит из:

1. **Крючок** (первая строка) — самая важная, видна в превью уведомления
2. **Тело** — основное содержание
3. **CTA** — призыв к действию
4. **Подпись** — бренд или имя

**Шаблон**:
```
[Эмодзи] Крючок — цепляющая первая строка

Основной текст сообщения.
Разбивай на короткие абзацы.

Каждый абзац — одна мысль.

[Ключевые пункты через эмодзи:]
✅ Пункт первый
✅ Пункт второй
✅ Пункт третий

*[Призыв к действию]*
Кнопка / ссылка / инструкция

— Kent | Ваш AI-ассистент
```

### 3. Типы рассылок

#### Информационная
```
📢 Важное обновление!

[Что изменилось/произошло — 2-3 предложения]

Что это значит для вас:
• [пункт 1]
• [пункт 2]

Подробности: [ссылка]
```

#### Продающая
```
🔥 [Оффер — что предлагается]

[Почему это выгодно — 1-2 предложения]

Условия:
✅ [условие 1]
✅ [условие 2]
✅ [условие 3]

⏰ [Ограничение: срок, количество]

👉 [CTA: как получить] — [ссылка]
```

#### Событийная
```
📅 [Название события]

🗓 [дата], ⏰ [время]
📍 [место/ссылка]

[Описание — зачем приходить]

Программа:
1. [пункт]
2. [пункт]
3. [пункт]

Регистрация: [ссылка]
Мест осталось: [число]
```

#### Контентная / полезная
```
💡 [Заголовок — полезная тема]

[Вводное — почему это важно]

[Основной контент — советы, инструкция, лайфхак]

Сохраняйте — пригодится ✨

[CTA: подписка, репост, вопрос]
```

#### Напоминание
```
⏰ Напоминаем!

[О чём напоминание — 1-2 предложения]

📅 Дата: [дата]
📍 Место: [место]

[Дополнительная информация]

До встречи! 👋
```

### 4. A/B варианты

По умолчанию для каждой рассылки создавай 2 варианта с разными подходами:

**Вариант A** — прямой, конкретный:
```
🎯 Скидка 30% на все услуги до пятницы

Только 3 дня — любая услуга со скидкой 30%.
Без промокодов, без условий.

Записаться: @manager_bot
```

**Вариант B** — эмоциональный, сторителлинг:
```
История: клиентка Мария пришла "просто попробовать"...

...а через месяц привела 5 подруг 😊

Сейчас у нас такая же возможность для вас — 
попробуйте любую услугу со скидкой 30%.

Только до пятницы 👉 @manager_bot
```

Покажи оба варианта клиенту. Спроси, какой отправлять. Или предложи отправить разным сегментам для A/B теста.

### 5. Длина сообщения

| Тип | Рекомендуемая длина | Максимум |
|-----|---------------------|----------|
| Короткая (уведомление) | 100-200 символов | 300 |
| Стандартная | 300-600 символов | 1000 |
| Длинная (контентная) | 600-1500 символов | 4096 |

**Правило**: Если сообщение превышает 1500 символов — предложи разбить на 2 сообщения или сократить. В Telegram длинные сообщения плохо читают.

### 6. Предпросмотр и подтверждение

Перед отправкой ВСЕГДА:

1. Покажи текст рассылки в финальном виде
2. Укажи формат (text/markdown/html)
3. Спроси подтверждение: "Отправить эту рассылку?"
4. Укажи количество получателей (если известно)

```
📢 Предпросмотр рассылки:

---
[текст рассылки]
---

📊 Получатели: все подписчики (~ 150 чел.)
📝 Формат: Telegram Markdown
🔤 Длина: 423 символа

Отправить? (да / нет / редактировать)
```

### 7. Отправка через OpenClaw

Используй `message send` для отправки:

```
message send --broadcast --format markdown --text "[текст рассылки]"
```

Параметры:
- `--broadcast` — массовая отправка всем подписчикам
- `--segment [тег]` — отправка сегменту (если CRM подключена)
- `--format markdown` — формат Telegram markdown
- `--schedule [datetime]` — отложенная отправка
- `--file [path]` — прикрепить файл/изображение

### 8. Сохранение рассылок

Каждая рассылка сохраняется в `workspace/broadcasts/`:

```markdown
---
date: 2026-04-10
type: продающая
variant: A
status: sent
recipients: 150
---

# Рассылка: Скидка 30%

## Текст (Вариант A — отправлен)

[текст рассылки]

## Текст (Вариант B — не использован)

[текст варианта B]

## Результаты
- Отправлено: 150
- Доставлено: 148
- Прочитано: 89 (60%)
- Клики: 12 (8%)
- Отписки: 1

## Заметки
- Вариант A показал лучший CTR
- Лучшее время — 18:00 вторника
```

**Именование файла**: `YYYY-MM-DD-slug.md`

Пример: `2026-04-10-skidka-30.md`

### 9. Сегментация (с CRM)

Если подключён скилл `crm-notes`:

- Отправка по тегам: "Разошли только VIP-клиентам"
- Персонализация: "Привет, {name}!" (подстановка из CRM)
- Исключения: "Всем, кроме тега `бывший_клиент`"

Если CRM не подключена — отправка всем подписчикам.

### 10. Расписание рассылок

Рекомендуемое время для Telegram-рассылок:

| День | Лучшее время (МСК) | Комментарий |
|------|---------------------|-------------|
| Пн | 10:00-11:00 | Начало рабочей недели |
| Вт | 11:00-13:00, 18:00-19:00 | Пиковая активность |
| Ср | 12:00-13:00 | Середина недели |
| Чт | 11:00-13:00, 18:00-19:00 | Хороший отклик |
| Пт | 10:00-12:00 | До "пятничного расслабления" |
| Сб | 11:00-13:00 | Для B2C |
| Вс | Не рекомендуется | Низкая вовлечённость |

**Частота**: Не чаще 2-3 раз в неделю. При ежедневных рассылках растут отписки.

## Граничные случаи

- **Пустой список получателей**: Предупреди: "Нет подписчиков для рассылки. Сначала соберите аудиторию"
- **Слишком частые рассылки**: Если последняя была менее 24 часов назад — предупреди: "Вчера уже была рассылка. Частые сообщения увеличивают отписки. Продолжить?"
- **Ссылки в тексте**: Проверь, что ссылки валидны (начинаются с https://, не битые). Рекомендуй укорачиватели (например, bit.ly) для отслеживания кликов
- **Эмодзи-перегруз**: Если в тексте больше 10 эмодзи — предложи сократить. Эмодзи хороши в меру, перебор выглядит спамно
- **Без CTA**: Каждая рассылка должна иметь CTA. Если клиент не указал — предложи добавить: "Без призыва к действию рассылка теряет 40% эффективности. Добавить?"
- **Чувствительный контент**: Не рассылай сообщения с ложной срочностью ("ПОСЛЕДНИЙ ШАНС!!!"), обещаниями гарантированного дохода, или вводящей в заблуждение информацией
- **Мультимедиа**: Если нужно отправить рассылку с изображением — сначала подготовь изображение (через social-drafts или вручную), затем прикрепи через `--file`
