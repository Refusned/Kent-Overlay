# 📄 Doc Generator — Генератор документов

Создание бизнес-документов для российского рынка. Запоминает реквизиты, подставляет данные, экспортирует в DOCX.

## Типы документов

### Деловые (для предпринимателей и фрилансеров)
| Команда | Документ | Что нужно от пользователя |
|---------|----------|--------------------------|
| "составь договор" | Договор оказания услуг / поставки / аренды | Стороны, предмет, цена, сроки |
| "выстави счёт" | Счёт на оплату | Покупатель, товары/услуги, суммы |
| "акт выполненных работ" | Акт | Номер договора, перечень работ, суммы |
| "составь КП" | Коммерческое предложение | Клиент, услуги, цены, кейс |
| "составь ТЗ" | Техническое задание | Описание задачи, требования, сроки |

### Юридические
| Команда | Документ | Что нужно |
|---------|----------|-----------|
| "претензия" | Досудебная претензия | Контрагент, нарушение, требования |
| "доверенность" | Доверенность (простая) | Доверитель, представитель, полномочия |
| "исковое" | Проект искового заявления | Стороны, обстоятельства, требования |

### Бухгалтерские
| Команда | Документ | Что нужно |
|---------|----------|-----------|
| "акт сверки" | Акт сверки взаиморасчётов | Контрагент, период, операции |
| "пояснительная записка" | Пояснительная к отчётности | Тип, причины, данные |

### Студенческие
| Команда | Документ | Что нужно |
|---------|----------|-----------|
| "титульный лист" | Титульный по ГОСТ | Вуз, кафедра, тема, ФИО |
| "список литературы" | Библиография по ГОСТ Р 7.0.5 | Список источников |

## Процесс работы

### 1. Сохранение реквизитов (один раз)
При первом использовании Kent спрашивает:
```
Для генерации документов мне нужны ваши реквизиты. 
Заполним один раз — буду использовать во всех документах.

Вы: [физлицо / ИП / ООО]?
```

Далее собирает и сохраняет в `workspace/documents/requisites.json`:
- Для физлица: ФИО, паспорт, адрес
- Для ИП: ФИО, ИНН, ОГРНИП, адрес, расчётный счёт, банк, БИК
- Для ООО: наименование, ИНН, КПП, ОГРН, адрес, расчётный счёт, банк, БИК, ФИО директора

### 2. Генерация документа
Пользователь: "Выстави счёт для ООО Ромашка за дизайн логотипа 25000 руб"

Kent:
1. Берёт реквизиты поставщика из `requisites.json`
2. Спрашивает недостающие данные контрагента (ИНН, адрес)
3. Генерирует текст документа
4. Показывает превью в Telegram
5. По подтверждению — экспортирует в DOCX и отправляет файлом

### 3. Нумерация
- Автоматическая нумерация: Счёт №001, №002, ...
- Сохраняется в `workspace/documents/counters.json`
- Формат: {тип}-{год}-{номер} → "С-2026-001"

### 4. Суммы прописью
- Автоматическая конвертация: 25 000 руб → "Двадцать пять тысяч рублей 00 копеек"
- НДС: если пользователь на ОСНО — рассчитывать и выделять 20%
- Если ИП на УСН → "Без НДС"

## Формат вывода

1. Сначала текстовое превью в Telegram (для проверки)
2. После подтверждения: файл .docx (через python-docx)
3. Опционально: .pdf (через reportlab)

## Шаблоны

Шаблоны хранятся как инструкции в этом SKILL.md, НЕ как файлы. Kent генерирует документ по стандартной структуре:

### Структура счёта
```
СЧЁТ на оплату №{номер} от {дата}

Поставщик: {наименование}, ИНН {инн}, КПП {кпп}
Банк: {банк}, БИК {бик}, Р/с {рс}, К/с {кс}

Покупатель: {наименование}, ИНН {инн}

| №  | Наименование | Кол-во | Ед. | Цена | Сумма |
|----|-------------|--------|-----|------|-------|
| 1  | {товар}     | {кол}  | шт  | {ц}  | {с}   |

Итого: {итого}
НДС: {ндс или "Без НДС"}
Всего к оплате: {всего} ({прописью})

Руководитель: ____________ / {ФИО} /
```

## Безопасность

- Реквизиты хранятся локально в workspace (зашифрованы at rest)
- В групповых чатах: НИКОГДА не показывать реквизиты
- Перед отправкой файла — ВСЕГДА показать превью и запросить подтверждение
- Не отправлять документы третьим лицам без явного указания

## Обработка ошибок

- Нет реквизитов → "Сначала заполним ваши реквизиты. Вы: физлицо, ИП или ООО?"
- Нет данных контрагента → "Для счёта нужен ИНН покупателя. Подскажите?"
- python-docx не установлен → "Нужна установка: pip install python-docx. Установить?"
