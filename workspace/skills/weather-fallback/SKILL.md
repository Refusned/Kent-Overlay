# Weather Fallback Skill

Используй этот skill, когда пользователь спрашивает о погоде, а основной путь недоступен или даёт сбой.

## Когда использовать

- weather skill недоступен
- локальный exec недоступен
- `wttr.in/<City>?format=3` не отвечает или возвращает ошибку
- нужен быстрый текущий прогноз без тяжёлых инструментов

## Основной способ

Запрашивай JSON через `web_fetch`:

- `https://wttr.in/<City>?format=j1`

После этого доставай из ответа `current_condition[0]`.

## Что отвечать пользователю

По умолчанию отвечай коротко:

- текущая температура
- ощущается как
- влажность
- давление
- видимость
- ветер и осадки, если они явно доступны

Пример формата:

```text
В Ижевске сейчас:
- 0°C
- ощущается как -5°C
- влажность 98%
- давление 1017 hPa
- видимость 2 км
```

## Правила

1. Не выдумывай значения, если JSON не прочитался надёжно.
2. Если short-format не работает, это не проблема, если `format=j1` вернул данные.
3. Если источник недоступен полностью, честно скажи об этом.
4. Для русских городов можно использовать латиницу в URL, если так стабильнее: `Moscow`, `Izhevsk`, `Kazan`.
5. Если пользователь просит прогноз, можно попробовать тот же JSON-ответ, но текущие условия приоритетнее.

## Заметка

Этот skill не заменяет основной weather skill, а страхует его отказ.
