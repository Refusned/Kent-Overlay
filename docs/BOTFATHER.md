# Настройка Telegram-бота через BotFather

Пошаговый чек-лист для создания и настройки Telegram-бота Kent через @BotFather.

---

## Шаг 1. Создание бота

1. Откройте Telegram и найдите бота **@BotFather**.
2. Отправьте команду `/newbot`.
3. Введите **отображаемое имя** бота. Примеры:
   - `Kent | Иван Петров`
   - `Kent для CoffeeCo`
   - `Kent AI Assistant`
4. Введите **username** бота (должен заканчиваться на `bot`). Примеры:
   - `ivan_kent_bot`
   - `coffeeco_kent_bot`
   - `kent_assistant_bot`
5. BotFather выдаст **токен**. Сохраните его -- он понадобится для `.env`.

> **Важно:** Никому не передавайте токен бота. Если он скомпрометирован, отзовите через `/revoke` в BotFather.

---

## Шаг 2. Установка имени

Если нужно изменить имя после создания:

1. Отправьте `/setname` в @BotFather.
2. Выберите бота из списка.
3. Введите новое имя.

**Рекомендации по имени:**
- Короткое и запоминающееся (до 30 символов).
- Содержит "Kent" для узнаваемости бренда.
- Можно добавить имя клиента или компании.

---

## Шаг 3. Установка username

Username нельзя изменить после создания. Выбирайте тщательно:

- Формат: `[клиент]_kent_bot` или `kent_[клиент]_bot`.
- Только латиница, цифры и подчеркивания.
- Должен заканчиваться на `bot`.

---

## Шаг 4. Установка описания (Description)

Описание видно пользователю при первом открытии бота (до нажатия "Начать").

1. Отправьте `/setdescription` в @BotFather.
2. Выберите бота.
3. Введите описание.

### Рекомендуемый текст (шаблон):

```
Kent -- ваш персональный AI-ассистент.

Я помогу вам с:
- Генерацией контента и текстов
- Управлением календарем и почтой
- Поиском информации в интернете
- Созданием изображений и презентаций
- Работой с файлами и документами
- Голосовыми сообщениями

Просто напишите или запишите голосовое -- я пойму.

Нажмите "Начать" для первого знакомства.
```

### Персонализированный вариант:

```
Kent -- персональный AI-ассистент для [Имя клиента].

Помогаю с бизнес-задачами, контентом и повседневными делами.
Понимаю текст и голос. Работаю 24/7.

Нажмите "Начать" чтобы познакомиться.
```

---

## Шаг 5. Установка краткого описания (About)

About-текст виден в профиле бота.

1. Отправьте `/setabouttext` в @BotFather.
2. Выберите бота.
3. Введите текст (до 120 символов).

### Рекомендуемый текст:

```
Персональный AI-ассистент. Контент, планирование, аналитика, голосовые команды.
```

### Варианты:

```
AI-ассистент Kent. Помогаю с бизнесом, контентом и задачами 24/7.
```

```
Kent -- умный помощник для бизнеса. Текст, голос, изображения, файлы.
```

---

## Шаг 6. Установка аватара

1. Отправьте `/setuserpic` в @BotFather.
2. Выберите бота.
3. Отправьте изображение.

### Рекомендации:

- **Файл:** `assets/kent-avatar.png` из репозитория kent-overlay.
- **Формат:** PNG или JPG.
- **Размер:** 512x512 пикселей (минимум 150x150).
- **Содержание:** логотип Kent или персонализированный аватар.
- Избегайте мелких деталей -- аватар отображается маленьким в списке чатов.
- Выбирайте контрастные цвета для читаемости.

---

## Шаг 7. Установка списка команд

1. Отправьте `/setcommands` в @BotFather.
2. Выберите бота.
3. Отправьте список команд в формате `command - описание`:

```
help - Справка по командам бота
lead - Создать или обработать лид
content - Генерация контента для соцсетей
faq - Часто задаваемые вопросы
brief - Составить бриф для задачи
home - Управление умным домом
draw - Сгенерировать изображение
slides - Создать презентацию
smm - Планирование SMM-публикаций
code - Помощь с кодом и скриптами
crm - Управление клиентской базой
broadcast - Массовая рассылка сообщений
settings - Настройки бота
feedback - Оставить отзыв или предложение
status - Текущий статус системы
```

> **Примечание:** Telegram поддерживает до 100 команд. Указанные 15 -- стандартный набор Kent.

### Минимальный набор (если не все интеграции включены):

```
help - Справка и список возможностей
content - Генерация контента
search - Поиск информации
settings - Настройки ассистента
status - Состояние сервисов
```

Добавляйте команды только для тех интеграций, которые реально настроены у клиента.

---

## Шаг 8. Отключение Privacy Mode

По умолчанию бот в групповых чатах видит только сообщения с команд (`/`) или упоминания. Для полноценной работы в группах нужно отключить Privacy Mode.

1. Отправьте `/setprivacy` в @BotFather.
2. Выберите бота.
3. Выберите **Disable**.

> **Когда отключать:** Если клиент планирует добавлять бота в групповые чаты, чтобы Kent мог читать все сообщения и участвовать в обсуждении.

> **Когда оставить включенным:** Если бот используется только в личных чатах, Privacy Mode не влияет на работу.

---

## Шаг 9. Получение токена

Если вы потеряли токен:

1. Отправьте `/mybots` в @BotFather.
2. Выберите бота.
3. Нажмите **API Token**.

Или сгенерируйте новый:

1. Отправьте `/revoke` в @BotFather.
2. Выберите бота.
3. Получите новый токен.

> **Важно:** После `/revoke` старый токен перестает работать. Обновите `.env` на сервере и перезапустите gateway.

---

## Шаг 10. Inline Mode (опционально)

Inline mode позволяет использовать бота из любого чата через `@username запрос`.

1. Отправьте `/setinline` в @BotFather.
2. Выберите бота.
3. Введите placeholder-текст, например: `Спросите Kent...`

### Когда полезен:
- Клиент хочет использовать Kent прямо из диалога с другим человеком.
- Быстрые запросы без переключения в чат с ботом.

### Когда не нужен:
- Бот используется только в личном чате.
- Клиент не знаком с inline-ботами.

---

## Итоговый чек-лист

После настройки убедитесь, что все пункты выполнены:

- [ ] Бот создан, username выбран
- [ ] Токен получен и сохранен
- [ ] Имя бота установлено
- [ ] Аватар бота установлен
- [ ] Описание (Description) заполнено
- [ ] Краткое описание (About) заполнено
- [ ] Список команд установлен (15 или меньше по набору интеграций)
- [ ] Privacy Mode настроен (Disable для групп / Enable для личного)
- [ ] Токен добавлен в `.env` на сервере
- [ ] Бот сопряжен через Control UI
- [ ] Тестовое сообщение `/start` -- бот отвечает

## Миграция бота

Если нужно перенести бота на новый сервер:

1. Скопируйте токен из `.env` старого сервера.
2. Остановите Kent на старом сервере:
   ```bash
   sudo systemctl stop kent-gateway
   ```
3. На новом сервере укажите тот же токен в `.env`.
4. Запустите Kent на новом сервере.

> **Важно:** Одновременно может работать только один экземпляр с одним токеном. Остановите старый перед запуском нового.
