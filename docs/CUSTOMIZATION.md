# Кастомизация Kent для каждого клиента

Руководство для оператора: как адаптировать Kent AI Assistant под конкретного бизнес-клиента.

---

## Обзор

Каждый экземпляр Kent разворачивается на отдельном VPS для конкретного клиента. Персонализация включает:

1. Настройку окружения (`.env`)
2. Персонализацию личности (SOUL.md)
3. Пользовательский профиль (USER.md)
4. Настройки BotFather
5. Выбор навыков и интеграций
6. Брендинг бота

---

## Что менять для каждого нового клиента

### Минимальный набор изменений

| Файл / настройка | Что менять | Обязательно |
|-------------------|-----------|-------------|
| `.env` | Токен бота, API-ключи | Да |
| `USER.md` | Информация о клиенте | Да |
| `SOUL.md` | Личность и стиль общения | Рекомендуется |
| `openclaw.json` | Список навыков | По необходимости |
| BotFather | Имя, описание, аватар бота | Да |

## Файл .env

Основной файл конфигурации. Создается автоматически при запуске `configure.sh`, но можно редактировать вручную.

### Обязательные переменные

```env
# Telegram
TELEGRAM_BOT_TOKEN=7000000000:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TELEGRAM_ADMIN_ID=123456789

# Codex (модели)
CODEX_API_KEY=sk-xxxxxxxxxxxxxxxx

# ChatGPT (генерация изображений)
CHATGPT_SESSION_TOKEN=eyJxxxxxxxxxxxxxxxx

# Основные настройки
BOT_NAME=Kent
CLIENT_NAME=Иван Петров
TIMEZONE=Europe/Moscow
LANGUAGE=ru
```

### Опциональные переменные

```env
# ElevenLabs TTS
ELEVENLABS_API_KEY=xxxxxxxxxxxxxxxx
ELEVENLABS_VOICE_ID=xxxxxxxxxxxxx

# Tavily Search
TAVILY_API_KEY=tvly-xxxxxxxxxxxxxxxx

# Google интеграции
GOOGLE_CLIENT_ID=xxxxxxxxxxxxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPx-xxxxxxxxxxxxxxxx
GOOGLE_REFRESH_TOKEN=1//xxxxxxxxxxxxxxxx

# Twitter/X
TWITTER_API_KEY=xxxxxxxxxxxxxxxx
TWITTER_API_SECRET=xxxxxxxxxxxxxxxx

# Yandex (Алиса / умный дом)
YANDEX_TOKEN=xxxxxxxxxxxxxxxx

# GitHub

```

## Запуск configure.sh для нового клиента

Скрипт `configure.sh` проведет вас через интерактивную настройку:

```bash
cd ~/kent-overlay
./configure.sh
```

### Что делает скрипт:

1. Запрашивает токен Telegram-бота
2. Спрашивает имя клиента и часовой пояс
3. Предлагает выбрать набор навыков из списка
4. Генерирует `.env` с заполненными полями
5. Создает шаблоны USER.md и SOUL.md
6. Проверяет валидность введенных токенов

### Повторный запуск

Можно запускать повторно -- скрипт предложит сохранить текущие значения или перезаписать:

```bash
./configure.sh --reconfigure
```

## Персонализация SOUL.md

Файл `SOUL.md` определяет характер, стиль общения и поведение ассистента.

### Расположение

```
kent-overlay/workspace/SOUL.md
```

### Структура файла

```markdown
# Личность

Ты -- Kent, персональный AI-ассистент [Имя клиента].

## Стиль общения
- Обращение на "вы" / "ты" (выбрать)
- Формальный / дружеский / деловой тон
- Краткие ответы / развернутые объяснения

## Специализация
- Бизнес и маркетинг
- [Добавить специфику клиента]

## Ограничения
- Не давать медицинских/юридических советов
- [Добавить ограничения клиента]

## Контекст клиента
- Сфера деятельности: [описание]
- Целевая аудитория: [описание]
- Ключевые продукты: [список]
```

### Примеры персонализации

**Для маркетолога:**

```markdown
Ты -- Kent, ассистент маркетолога Анны Смирновой.
Обращайся на "ты". Тон -- дружеский, но профессиональный.
Специализируешься на контент-маркетинге, SMM, аналитике.
Всегда предлагай конкретные идеи, а не общие советы.
При генерации контента учитывай целевую аудиторию: 
женщины 25-45, интересующиеся здоровым образом жизни.
```

**Для предпринимателя:**

```markdown
Ты -- Kent, бизнес-ассистент Дмитрия Козлова.
Обращайся на "вы". Тон -- деловой, лаконичный.
Специализируешься на управлении проектами, финансовом анализе, 
стратегическом планировании.
Всегда считай ROI при оценке идей.
Бизнес клиента: сеть кофеен в Москве, 12 точек.
```

**Для блогера:**

```markdown
Ты -- Kent, креативный ассистент блогера Марии.
Обращайся на "ты". Тон -- легкий, творческий, вдохновляющий.
Помогай генерировать идеи для контента, писать посты,
придумывать заголовки и хештеги.
Платформы: Instagram, Telegram, YouTube.
Ниша: путешествия и lifestyle.
```

## Добавление и удаление навыков

Навыки (skills) управляются через файл конфигурации `openclaw.json`.

### Просмотр доступных навыков

```bash
cat ~/kent-overlay/config/openclaw.json | python3 -m json.tool
```

### Отключение навыка

Установите `"enabled": false` для ненужного навыка:

```json
{
  "skills": {
    "smart_home": {
      "enabled": false
    },
    "github": {
      "enabled": false
    }
  }
}
```

### Добавление навыка

Добавьте запись в секцию `skills` и установите `"enabled": true`. Убедитесь, что все необходимые переменные окружения для навыка указаны в `.env`.

### Типичные наборы навыков по типу клиента

**Маркетолог / SMM-менеджер:**
- content (генерация контента)
- image (генерация изображений)
- social (публикация в соцсети)
- calendar (планирование)
- email (рассылки)
- search (исследования)
- tts (озвучка)

**Предприниматель / CEO:**
- email (корреспонденция)
- calendar (расписание)
- drive (документы)
- search (аналитика)
- pptx (презентации)
- lead (CRM)

**Разработчик:**
- github (код)
- coder (программирование)
- search (документация)
- email (коммуникации)

**Блогер / Контент-мейкер:**
- content (тексты)
- image (визуал)
- social (публикации)
- tts (озвучка)
- stt (транскрипция)
- browser (ресерч)

## Настройка интеграций для клиента

Каждая интеграция требует отдельной настройки. Подробности -- в [INTEGRATIONS.md](INTEGRATIONS.md).

### Быстрый выбор интеграций

Ответьте на вопросы вместе с клиентом:

1. **Нужна ли работа с email?** -- Настройте Gmail.
2. **Нужен ли календарь?** -- Настройте Google Calendar.
3. **Работа с документами?** -- Настройте Google Drive.
4. **Публикации в соцсети?** -- Настройте нужные платформы.
5. **Голосовые ответы?** -- Настройте ElevenLabs.
6. **Умный дом?** -- Настройте Yandex.
7. **Работа с кодом?** -- Настройте GitHub.

## Мультиклиентная настройка

### Принцип: один VPS -- один клиент

Каждый клиент получает отдельный VPS с полностью изолированным экземпляром Kent. Это обеспечивает:

- **Безопасность:** данные клиентов не пересекаются
- **Стабильность:** проблемы одного экземпляра не влияют на другие
- **Простоту управления:** независимое обновление и настройка
- **Масштабируемость:** легко добавлять новых клиентов

### Процесс подключения нового клиента

1. Заказать VPS (4GB RAM, 2 vCPU, 40GB SSD).
2. Выполнить [DEPLOYMENT.md](DEPLOYMENT.md) от начала до конца.
3. Настроить `.env`, `USER.md`, `SOUL.md` под клиента.
4. Выбрать и настроить интеграции.
5. Настроить бота в BotFather ([BOTFATHER.md](BOTFATHER.md)).
6. Передать клиенту [CLIENT-QUICKSTART.md](CLIENT-QUICKSTART.md).

### Управление несколькими серверами

Рекомендуется вести таблицу:

| Клиент | IP сервера | Бот | Статус | Дата деплоя |
|--------|-----------|-----|--------|-------------|
| Иван Петров | 1.2.3.4 | @ivan_kent_bot | Активен | 2026-01-15 |
| Анна Смирнова | 5.6.7.8 | @anna_kent_bot | Активен | 2026-02-01 |

## Брендинг бота

### Имя бота

Рекомендуемые форматы:
- `Kent | [Имя клиента]` -- например, `Kent | Иван Петров`
- `Kent для [Компания]` -- например, `Kent для CoffeeCo`
- Или любое имя по желанию клиента

### Аватар бота

1. Используйте стандартный аватар Kent (из `assets/kent-avatar.png`).
2. Или создайте персонализированный аватар с брендом клиента.
3. Установите через BotFather: `/setuserpic`.

Рекомендуемые параметры аватара:
- Формат: PNG или JPG
- Размер: 512x512 px
- Без мелких деталей (плохо видно в списке чатов)

### Описание бота

Устанавливается через BotFather (`/setdescription`). Пример:

```
Kent -- ваш персональный AI-ассистент.

Помогу с контентом, планированием, аналитикой и повседневными задачами. 
Пишите текстом или голосом -- я пойму.

Создан специально для [Имя клиента / Компания].
```

### Краткое описание (About)

Устанавливается через BotFather (`/setabouttext`). Пример:

```
Персональный AI-ассистент на базе Kent. 
Контент, планирование, аналитика, умный дом.
```

## Обновление Kent у клиента

### Обновление кода

```bash
cd ~/kent-overlay
git pull origin main
pnpm install
sudo systemctl restart kent-gateway
```

### Обновление конфигурации

```bash
# Отредактируйте нужные файлы
nano ~/kent-overlay/workspace/SOUL.md
nano ~/kent-overlay/.env

# Перезапустите gateway
sudo systemctl restart kent-gateway
```

### Проверка после обновления

1. Отправьте `/help` боту -- должен ответить.
2. Отправьте голосовое сообщение -- должен распознать.
3. Проверьте логи: `journalctl -u kent-gateway -n 50`.
